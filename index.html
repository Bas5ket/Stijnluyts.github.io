Het verder uitrollen van ClearPass

Voor mijn stageopdracht bij Politiezone Geel-Laakdal-Meerhout moest ik de ClearPass Network Access Control (NAC) verder uitrollen.

Hiervoor maakte ik gebruik van Microsoft Intune om de nodige configuraties op de laptops van de medewerkers te implementeren. De ClearPass-oplossing was voor mijn stage al gedeeltelijk opgezet en geconfigureerd, maar ik moest deze verder aanvullen en optimaliseren.

Daarnaast heb ik de Aruba-switches geconfigureerd, zodat deze konden communiceren met de ClearPass-server. Dit maakte het mogelijk om de laptops correct te laten authenticeren via het netwerk.

Tot slot heb ik een handleiding opgesteld die intern gebruikt kan worden. Hierin wordt toegelicht wat ik heb uitgevoerd, welke keuzes ik heb gemaakt, en het belangrijkst, welke stappen er in de toekomst nog genomen kunnen worden.

Zabbix Migratie

Tijdens mijn stage bij CISA heb ik het Zabbix-monitoringsysteem gemigreerd. Deze migratie was nodig omdat de bestaande systemen nog draaiden op een verouderde versie.

Ik was verantwoordelijk voor het volledige project en moest het zelfstandig leiden.
Eerst zette ik bij de klanten van CISA een Zabbix-proxy op, configureerde deze met een IP-adres en hostname, en zorgde ervoor dat de proxy verbinding kon maken met de centrale proxy van CISA. Hiervoor maakte ik een firewallregel aan zodat de communicatie via een specifieke poort mogelijk was.
Vervolgens paste ik de configuratie aan op alle servers die gemonitord moesten worden, zodat deze via de nieuwe proxy liepen.

Tijdens dit project heb ik geleerd hoe ik een plan van aanpak opstel, gestructureerd werk lever en alles zorgvuldig documenteer.

Netwerkopstelling Rusthuis

Tijdens mijn allereerste projectweek werkten we aan een concept voor een modern rusthuis. Het doel was om ervoor te zorgen dat elke bewoner beschikte over een stabiele wifi-verbinding en de mogelijkheid had om te casten via Chromecast. Daarnaast ontwierpen we een netwerkconcept met gebruik van routers en switches.

Ik was verantwoordelijk voor het opstellen van het netwerkconcept. Hiervoor stelde ik een ontwerp voor met drie routers en vijf switches, om te simuleren hoe het netwerk op grotere schaal zou functioneren. In ons plan kreeg elke kamer een eigen subnet met acht vrije IP-adressen, terwijl alle wifi-apparaten in één gedeeld subnet zaten. Op deze manier kon elke kamer een eigen Chromecast gebruiken zonder dat andere bewoners toegang kregen tot elkaars apparaten.

Door dit project leerde ik hoe je een netwerk opzet op grotere schaal. Tot dan toe hadden we vooral gewerkt met eenvoudige netwerken met één router en twee switches. Dit project gaf me de kans om breder na te denken en een realistischer netwerkontwerp te maken.

Project 4.0

Voor Project 4.0 moest ons team een seating app ontwikkelen voor Elision. Het doel van deze webapplicatie was om medewerkers de mogelijkheid te geven om een werkplek te reserveren op kantoor. Deze behoefte ontstond na de coronaperiode, aangezien het bedrijf zijn personeel minstens twee dagen per week op kantoor wilde laten werken om de soft skills te versterken.

Als enige student uit de afstudeerrichting Cloud & Cyber Security (CCS) in het team was ik verantwoordelijk voor het opzetten van de Amazon Web Services (AWS)-omgeving waarin de applicatie gehost zou worden. Hiervoor stelde ik een kostenraming op en werkte ik een cloudarchitectuur uit, die ik ook moest verdedigen tegenover het team en de begeleiders om aan te tonen waarom deze opzet noodzakelijk was.

Daarnaast heb ik Jenkins moeten leren gebruiken om een CI/CD-pipeline op te zetten. Deze pipeline zorgt ervoor dat zodra de applicatie naar Bitbucket wordt gepusht, deze automatisch wordt gedeployed naar de hostingomgeving. Omdat we in dit project verplicht waren om met Bitbucket te werken, kon de standaard GitLab-pipeline niet worden gebruikt. Daarom koos ik voor Jenkins als alternatief.

Tijdens dit project heb ik vooral geleerd hoe belangrijk goede communicatie is binnen een multidisciplinair team. Aangezien ik samenwerkte met studenten die gespecialiseerd zijn in andere domeinen van IT, moest ik duidelijk kunnen uitleggen waarom we bijvoorbeeld gekozen hebben voor Amazon RDS (Relational Database Service) een technologie waarmee zij minder vertrouwd waren. Omgekeerd legden zij mij hun keuzes en werkwijzen uit, zodat ik indien nodig de AWS-omgeving kon aanpassen.

Tot slot heb ik ook veel bijgeleerd over het werken met Jenkins en het implementeren van CI/CD buiten de vertrouwde tools die we normaal in onze opleiding gebruiken.

Project hosting

Voor het project Hosting Platform moesten we een systeem ontwikkelen waarop gebruikers hun eigen applicaties tijdelijk konden hosten om te testen. Het platform moest toegankelijk zijn voor anderen zodat zij de gehoste applicaties ook konden testen.

Binnen dit project was ik voornamelijk verantwoordelijk voor de backend. Mijn taken bestonden onder andere uit het automatiseren van het opstarten van virtuele machines (VM’s) en ervoor zorgen dat alles op de achtergrond vlot verliep. Dit hield in dat ik de nodige configuraties moest voorzien zodat elke VM automatisch een IP-adres kreeg via DHCP, zodat gebruikers verbinding konden maken met hun applicatie.

Daarnaast heb ik Ansible klaargezet om ervoor te zorgen dat, zodra een nieuwe VM werd opgestart, automatisch de juiste configuraties werden uitgerold. Dit maakte het beheer van meerdere VM’s efficiënter en consistenter.

Een andere belangrijke taak was het opzetten van een beveiligde file transfer-mogelijkheid, zodat gebruikers hun projectbestanden konden uploaden naar de VM. Hierbij moest ik er ook voor zorgen dat gebruikers geen toegang hadden tot andere delen van het systeem, om de veiligheid te garanderen.

Tot slot heb ik een Apache-webserver geïnstalleerd en geconfigureerd zodat de geüploade applicaties effectief gehost konden worden en toegankelijk waren via de browser.

Tijdens dit project heb ik geleerd om vooraf goed na te denken over welke tools en technologieën nodig zijn om een project succesvol op te zetten. Zo was het inzetten van Ansible een logische en waardevolle keuze voor het beheren van de configuraties, maar het voorzien van een veilige file transfer was iets dat ik in eerste instantie over het hoofd had gezien. Dit project heeft me geholpen om theoretische kennis, zoals het werken met Ansible of het opstarten van een VM vanuit een template, in een realistische context toe te passen.
